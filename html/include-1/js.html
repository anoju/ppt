Part3-1JavaScript

JavaScriptJS
자바스크립트는 ‘웹페이지에 생동감을 불어넣기 위해’ 만들어진 프로그래밍 언어입니다. 자바스크립트로 작성한 프로그램을 스크립트(script) 라고 부릅니다. 스크립트는 웹페이지의 HTML 안에 작성할 수 있는데, 웹페이지를 불러올 때 스크립트가 자동으로 실행됩니다.

스크립트는 특별한 준비나 컴파일 없이 보통의 문자 형태로 작성할 수 있고, 실행도 할 수 있습니다. 이런 관점에서 보면 자바스크립트는 자바(Java)와는 매우 다른 언어라고 할 수 있습니다.

현재 가장 많이 사용하는 개발언어(redmonk사이트 2021년기준)로

  1. 1. 다양한 용도

    자바스크립트는 웹 프론트엔드 개발뿐만 아니라 서버 사이드 개발, 데스크탑 애플리케이션, 모바일 애플리케이션 등 다양한 분야에서 사용될 수 있습니다.

  2. 2. 높은 상호성

    자바스크립트는 다양한 플랫폼에서 작동하며, 다른 언어로 작성된 코드와 상호작용할 수 있습니다.

  3. 3. 쉬운 학습 곡선

    자바스크립트는 배우기 쉬우며, 대부분의 브라우저에서 즉시 실행됩니다. 이를 통해 빠른 프로토타이핑이 가능합니다.

  4. 4. 강력한 라이브러리와 프레임워크

    자바스크립트는 jQuery, React, Vue.js, AngularJS 등 다양한 라이브러리와 프레임워크를 제공합니다.

Part3-3JavaScript 단점

  1. 1. 브라우저 호환성

    각 브라우저의 다른 자바스크립트 엔진으로 인해 브라우저 호환성 문제가 발생할 수 있습니다.

  2. 2. 보안 취약점

    자바스크립트는 클라이언트 측에서 실행되므로, 보안 취약점이 존재할 수 있습니다.

  3. 3. 상대적으로 느린 실행 속도

    자바스크립트는 인터프리터 언어이기 때문에, 컴파일 언어보다 상대적으로 느린 실행 속도를 보입니다.
    이러한 문제를 극복하기 위해 JIT 컴파일러 등 다양한 방법이 사용되고 있습니다.

인터프리터 언어
인터프리터 언어는 소스 코드를 한 줄씩 읽어들이면서 실행하는 프로그래밍 언어입니다.
이와 대조적으로 컴파일러 언어는 소스 코드를 먼저 컴파일하여 실행 파일을 만들고, 이후에 실행하는 방식입니다. 인터프리터 언어는 코드 실행 시간에 직접 소스 코드를 해석하여 실행하기 때문에, 컴파일러 언어에 비해 실행 속도가 느릴 수 있습니다.
그러나 인터프리터 언어는 코드 수정 및 디버깅이 간편하며, 실행 환경에 따른 호환성 문제가 적은 장점이 있습니다. 자바스크립트는 대표적인 인터프리터 언어 중 하나입니다. 자바스크립트 코드는 브라우저 내에서 바로 실행되며,
브라우저마다 다르게 해석될 수 있는 문제를 방지하기 위해 ECMA 스크립트 표준이 제정되었습니다.

Part3-4ECMA스크립트

ECMAScriptES
ECMA 스크립트는 JavaScript 언어의 표준 규격을 정의하는 기술적 규격서입니다. ECMA 국제 표준화 기구(Ecma International: 전자 정보 통신 기술에 대한 유럽 컴퓨터 제조자 협회)에서 개발되었으며, ECMA-262 표준를 따르고 있습니다. Ecma internatinal은 정보통신기술(ICT), 전자제품(CE)를 위한 국제 표준 기구이다. ECMAScript의 언어 규격인 ECMA-262외에도 C#, C++, Dart, JSON 등 여러 시스템을 위한 표준을 책임지고 있습니다.
자바스크립트를 개발한 Netscape가 더 향상된 표준화를 위해 기술 규격을 Ecma에 제출했고, 그에 따라 Ecma가 새롭게 제정한 표준이 바로 ECMA-262입니다. ECMA-262는 규격이며, ECMAScript는 ECMA-262에 의해 표준화된 자바스크립트의 새로운 이름입니다.

Part3-6타입스크립트

TypeScriptTS
마이크로소프트에서 구현한 JavaScript의 슈퍼셋(Superset) 프로그래밍 언어. 확장자로는 .ts를 사용하며, 컴파일의 결과물로 JavaScript 코드를 출력한다. 최종적으로 런타임에서는 이렇게 출력된 JavaScript 코드를 구동시키게 됩니다.
TypeScript라는 이름답게 정적 타입을 명시할 수 있다는 것이 순수한 자바스크립트와의 가장 큰 차이점입니다. 덕분에 개발 도구(IDE나 컴파일러 등)에게 개발자가 의도한 변수나 함수 등의 목적을 더욱 명확하게 전달할 수 있고, 그렇게 전달된 정보를 기반으로 코드 자동 완성이나 잘못된 변수/함수 사용에 대한 에러 알림 같은 풍부한 피드백을 받을 수 있게 되므로 순수 자바스크립트에 비해 어마어마한 생산성 향상을 꾀할 수 있습니다. 즉, '자바스크립트를 실제로 사용하기 전에 있을만한 타입 에러들을 미리 잡는 것' 이 타입스크립트의 사용 목적입니다.

Part3-7타입스크립트 장점과 단점

타입스크립트는 JavaScript에 타입을 추가한 언어로, 정적 타입을 지원하여 코드의 안정성을 높이고 유지보수성을 높일 수 있습니다. 이에 따라, 타입스크립트의 장점과 단점은 다음과 같습니다.
장점
  1. 타입 지정으로 인한 코드 안정성 향상: 타입스크립트는 변수, 함수, 인터페이스 등에 타입을 지정하여 코드 안정성을 높입니다. 이를 통해, 런타임에 발생하는 에러를 줄일 수 있습니다.
  2. 유지보수성 향상: 타입스크립트는 타입 지정으로 인해 코드의 가독성이 향상되고, IDE에서 자동완성 및 코드 검증 등의 기능을 지원합니다. 이를 통해 코드 유지보수성을 향상시킬 수 있습니다.
  3. 객체지향 프로그래밍 지원: 타입스크립트는 클래스, 인터페이스, 상속 등의 객체지향 프로그래밍 패턴을 지원하여 코드의 재사용성을 높입니다.
  4. 대규모 프로젝트에 적합: 타입스크립트는 대규모 프로젝트에 적합한 언어입니다. 타입 지정으로 인해 코드의 안정성을 높일 수 있고, 코드 유지보수성이 높아집니다.
단점
  1. 학습 곡선이 높음: 타입스크립트는 JavaScript보다 복잡한 언어로, 학습 곡선이 높을 수 있습니다.
  2. 컴파일 시간이 느릴 수 있음: 타입스크립트는 코드를 컴파일하여 JavaScript로 변환해야 하므로, 컴파일 시간이 JavaScript보다 느릴 수 있습니다.
  3. 개발자 수가 적을 수 있음: 타입스크립트를 사용하는 개발자 수가 JavaScript보다 적기 때문에, 필요한 지식과 경험을 가진 개발자를 구하기 어려울 수 있습니다.
  4. 타입 에러 발생 가능성: 타입 지정을 하지 않은 변수, 함수 등이 존재할 경우, 타입 에러가 발생할 수 있습니다. 따라서, 코드 작성 시에는 타입 지정에 신경써야 합니다.
